TMEX File Operations API Overview Overview Group

The TMEX File Operations API layer functions do file and directory operations. The ROM pattern for the desired 1-Wire device must already be in the internal eight-byte buffer before any of the File Operations API functions can be called. This can be accomplished by directly writing to the internal buffer by using the TMRom API call or by using one of the Network functions TMFirst, TMNext, TMFirstAlarm or TMNextAlarm. This constraint enables all of these API calls to be multi-drop compatible. 'multi-drop compatible' refers to successful operation even though there are more than one device on a single 1-Wire network.
